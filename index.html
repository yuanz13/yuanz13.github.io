DreamHouse - a route planner website for house buyers

Buying a house can cost a fortune, we all want to find the house that suits us best from hundreds of houses for sale in the same area.

So, we search on Zillow and are provided with a list of houses. We can probably narrow down the candidates to around 10 houses by checking the webpages. And then, we need to physically go to the houses and and find out which one is the "dream house".

We drag our agent and hope to visit the candidate houses in one trip. But..., what is the best route in this scenario since there are so many stops?

I built a website just to fill this need. I used the Django framework with Bootstrap for front end, handled user registration using AllAuth (connected to a MySQL database), calculated the optimized route using a simulated annealing algorithm (or an advanced google map api call, which is faster). The optimized route is displayed on the webpage and also sent to the user's email address using the gmail SMTP service. I built a Docker image for this application and deployed the website on google cloud Kubernetes. (http://34.72.138.106/)

Step 1: sign up

First time users are required to sign up. The optimized route will be sent to the user registered email address for future reference.

Step 2: Search for houses

Enter the city name to search for houses. The results are retrieved from Zillow.

Step 3: Select the houses to visit

Select the houses to visit, enter the start and end addresses for the trip, then click "Get Route"

Step 4: Check the route

The optimized is displayed in a google map window; Details of each segment are also shown. Click on "Email me the route" to send this route to the user's registered email address.

Diffusiaon MRI + AI

Diffusion kurtosis is an important physical property that characterizes tissue structural complexity. It is closely related to pathology and can be used to assess tumor grade and malignancy noninvasively.

A novel multi-dimensional diffusional kurtosis imaging technique (md-DKI) provides clean measurements of tissue kurtosis from microanisotropy (MKA) and isotropic heterogeneity (MKI), which are indistinguishable in conventional DKI techniques. The micro fractional anisotropy (µFA) of tissue is also provided, which cannot be measured in conventional DKI either. However, md-DKI requires longer data acquisition and image processing time, and have not been evaluated for clinical diagnosis.

I have applied machine learning techniques in md-DKI to reduce the scan time, speed up image processing, and improve tumor classification. Please refer to the following publications for more details: Yuan Zheng, Tao Feng, Acceleration of multidimensional diffusion MRI data acquisition and post-processing using convolutional neural networks, ISMRM 2020: 4329 Li, S., Zheng, Y., et al. Glioma grading, molecular feature classification, and microstructural characterization using MR diffusional variance decomposition (DIVIDE) imaging. Eur Radiol (2021). https://doi.org/10.1007/s00330-021-07959-x

The Convolutional Neural Network (CNN)

md-DKI takes long scan time since it requires image acquisition at multiple diffusion encodings. I built a CNN that can generate the md-DKI parameters with only half of the diffusion encodings (reduced to 40 from 80), thereby saving half of the scan time. The image processing time for generating the md-DKI parameter maps are also reduced from ~30 min to ~2 s.

The CNN has 4 convolution layers (blue) and 4 deconvolution layers (green). The number of channels is first expanded and then gradually shrinked to 1, while the image dimension is kept at 112 × 112 throughout. Leaky-RELU was applied to all layers after convolution except the output layer, which is followed by conventional RELU activation.

Training

Ten-fold crossvalidation are applied in network tuning. After 200 epochs, both traing and validation losses became stable and there is no major difference between them.

Parametric Maps

The µFA maps of a meningioma case generated using the conventional method (80 encodings) and the CNN (40 encodings) were similar and shown in a) and b) respectively. In the 9 × 9 ROI centered on the tumor, µFA = 0.73 ± 0.05 and 0.70 ± 0.06 in a) and b).

Support Vector Machine (SVM) classifier

Low-grade gliomas (LGG) and high-grade gliomas (HGG) are shown as blue and red dots. Single predictor decision boundaries using the conventional MKT, or the novel parameter MKA, MKI are shown as dotted line in cyan, purple, and green. A dual-predictor (MKA and MKI) SVM is also build for the classification of LGG and HGG, and the decision boundary is shown as the black solid line. The respective sensitivity and specificity are 84.8% and 82.4% for grading. A linear kernel is used here to avoid overfitting. Performance of the classifier can definitely be improved by using a nonlinear kernel as more data are collected.

Cardiac cine MRI + AI

Cardiac cine MRI acquires a dynamic series of the popping heart, and is currently the gold standard for assessing myocardial function. It is typically acquired under multiple breatholds for whole heart coverage, which inevitably adds patient discomfort and complexity in the scan workflow.

In order to accelerate the scan, compressed sensing MRI are proposed, which undersamples the so-called “k-space” to save scan time. However, it requires iterative optimization for image reconstruction, which can take minutes.

To solve this problem, I trained an RNN network for real-time reconstruction of the dynamic image series from highly undersampled data.

The Cascade Convolutional Recurrent Neural Network

This neural network contains 5 iterations (superscripts) and 5 layers (subscripts). Multiple types of units are involved, including bidirectional convolutional recurrent units (BCRNN), convolutional recurrent units (CRNN), convolutional layer(CNN), and data consistency layer (DC). Residual connections are also adapted to make training more effective.

The training inputs are retrospectively 8-folder undersampled dynamic cardiac images, and the outputs are images reconstructed from the fully sampled data.

Structural Similarity Index during training

The structural similarity of the validation images increases during training, and plateaued at ~0.98 for epoch > 200.

Image comparison

The network input image series from 8× undersampled data is shown in a). The network reconstruction results are shown in b), and the ground truth (fully sampled data) are shown in c). The network reconstructed images have similar quality to the ground truth. The bluriness and artifacts in the original undersampled images are effectively removed.

Tissue longitudinal relaxiation time mapping + AI

Tissue longitudinal relaxiation time constant (referred to as "T1" in MRI jargon) mapping is an important magnetic resonance quantitative imaging technique, which has found many clinical applications, such as diagnosis of myocardial fibrosis.

Current myocardium T1 mapping method requires the patient to perform a 11-heartbeat breathold. Eight images with different inversion recovery times are collected, and T1 maps are generated based on a 3-parameter fit.

I built a fully-connected neural network to predict the T1 values using only 5 measurements, which can be acquired within 5 heartbeats and greatly reduces the patient breathold time.
